# 作業完了

作業完了後の一連の処理を実行してください。

## 処理フロー

### 1. 変更確認
- `git status` で変更ファイルを確認
- `git diff` で変更内容を確認

### 2. テスト実行
- `./gradlew clean build` でビルド
- `./gradlew integrationTest` で統合テスト

### 3. コミット
- 変更サマリーとコミットメッセージ案を提示
- ユーザー確認後にコミット実行

### 4. Push & PR作成
- `git push -u origin <branch>` でリモートにプッシュ
- `gh` コマンドの存在確認
  - **ghがある場合**: `gh pr create` でPR作成
  - **ghがない場合**: PR情報（タイトル、概要、変更内容）を出力して終了

## 注意事項

- 各フェーズでユーザーに確認を取ってから次へ進む
- テスト失敗時は原因を報告して停止
